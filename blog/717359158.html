nginx实验六

本文最后更新于：2023年10月22日 中午

调试输出

（1）采用命令下载第三方模块echo-nginx-module
（2）采用命令解压缩v0.60.tar.gz和nginx-1.10.3.tar.gz
（3）采用命令将echo-nginx-module-0.60移至echo-nginx-module目录。
（4）采用命令进入目录nginx-1.10.3

cd nginx-1.10.3

（5）执行命令生成makefile，并且添加echo-nginx-module模块
（6）执行命令生成可执行的二进制文件

​ make && make install

（7）采用命令备份原来的nginx可执行的二进制文件
（8）采用命令将新编译的nginx可执行的二进制文件复制到nginx的安装目录sbin
（9）修改nginx的主配置文件
（10）采用命令重新启动nginx
（11）打开浏览器进行测试，如果成功看到如下表示成功：

This is an echo module.

网页压缩传输

（1）准备好一个js及CSS文件
（2）修改index.html，引入js、CSS文件

vim index.html

（3）打开nginx的主配置文件，添加如下两行代码，启用gzip模块
（4）打开浏览器，按F12键打开开发者模式，若成功看到Content-Encoding:gzip表示成功启用网页压缩传输。

F12–>network–>Headers–>ResponseHeaders

基于rewrite的重写

（１）打开nginx的主配置的主配置文件，注释掉原来的默认虚拟主机。
（２）添加名为www.xp.com的虚拟主机
（３）在html目录下创建一名为xp-default的目录
（４）采用命令创建一测试页面
（５）重启nginx后，打开浏览器，访问一个不存在的页面并查看结果

Welcome to the default html. Xu Pan.

基于rewrite的重定向

（１）打开nginx的主配置的主配置文件，添加名为www.xp.net的虚拟主机：
（２）在html目录下创建一名为www.xp.net的目录，再在此目录下创建的一个img目录
（３）上传4张图片至img目录，图片名称为1.jpg，2.jpg，3.png，4.png
（４）重启启动nginx
（５）打开浏览器，进行测试，若能够成功看到相应的图片，表示重定向配置成功。

输入http://www.xp.net/img-1.jpg

输入http://www.xp.net/img-3.png

防盗链配置

（１）打开nginx的主配置的主配置文件，添加名为www.xp5.com和www.img.com的虚拟主机：
（２）在html/www.img.com中建立子目录img，同时上传一些图片到该目录

cd /usr/local/nginx/html

mkdir www.img.com

cd www.img.com

上传图片

（３）在网站www.img.com中建立测试页面index.html

vim index.html

（４）在网站www.xp5.com中盗用www.img.com中的图片，建立index.html，输入如下代码：

vim index.html

（５）打开浏览器，分别输入进行访问，发现www.xp5.com盗链成功
（６）修改www.img.com对应的配置文件，实现防盗链
（７）重新启动nginx

cd /usr/local/nginx/sbin ./nginx -s reload

（８）打开浏览器进行测试，如果发现看不到图片，则防盗链成功。
